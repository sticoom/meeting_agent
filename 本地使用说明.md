# 本地使用说明

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置 API Key

**方式一：使用 Secrets 文件（推荐）**

1. 复制 `.streamlit/secrets.toml.example` 为 `.streamlit/secrets.toml`
2. 编辑文件，填入您的 API Keys：
   ```toml
   GITHUB_TOKEN = "your_github_token"  # 可选，用于 GitHub 模式
   GITHUB_OWNER = "your_username"        # 可选，用于 GitHub 模式
   GITHUB_REPO = "your_repository_name"   # 可选，用于 GitHub 模式
   GLM_API_KEY = "your_glm_api_key"   # 必需，用于生成会议纪要
   ```

**方式二：在应用界面输入（临时）**

如果不想配置 secrets 文件，可以在应用启动后在侧边栏输入 GLM API Key。

### 3. 准备会议文件

将会议相关文件放入 `inputs/` 目录：

```
inputs/
├── 20260309_录音转写.docx    # 录音转写文件
└── 20260309_手写重点.txt      # 手写重点文件（可选）
```

**文件命名建议：**
- 转写文件：`日期_录音转写.docx` 或 `日期_会议录音文字版.docx`
- 手写重点：`日期_手写重点.txt` 或 `日期_会议纪要手写粗糙版.txt`

### 4. 启动应用

```bash
# Windows
启动小程序.bat

# 或直接运行
streamlit run app.py
```

### 5. 生成会议纪要

1. 应用启动后，会自动访问 http://localhost:8501
2. 在「📝 纪要生成」标签页：
   - 系统会自动读取 `inputs/` 目录中的最新文件
   - 点击「🚀 一键生成会议纪要」按钮
3. 等待 1-2 分钟，GLM-4 API 生成会议纪要
4. 生成的纪要自动保存到 `outputs/` 目录
5. 在「📊 结果展示」标签页查看结果

## 功能特性

### ✅ 自动文件读取
- 自动扫描 `inputs/` 目录
- 识别录音转写文件和手写重点文件
- 按修改时间选择最新文件

### ✅ 智能生成
- 使用 GLM-4 模型生成会议纪要
- 自动应用 skill.md 中的所有规则
- 自动读取和应用 reference 文件

### ✅ 自动保存
- 生成的纪要自动保存到 `outputs/` 目录
- 文件名格式：`管理周会纪要_YYYYMMDD_HHMMSS.md`
- 支持一键复制和下载

### ✅ Reference 自动更新
- 上传最终版纪要时自动更新 reference 文件
- 支持本地和 GitHub 模式

## 使用示例

### 示例 1：首次使用

```
1. 将会议文件放入 inputs/
   inputs/20260309_录音转写.docx
   inputs/20260309_手写重点.txt

2. 启动应用
   启动小程序.bat

3. 点击「🚀 一键生成会议纪要」

4. 等待生成完成（约 1-2 分钟）

5. 在 outputs/ 目录查看结果
   outputs/管理周会纪要_20260309_143025.md
```

### 示例 2：使用最终版更新 Reference

```
1. 在「📊 结果展示」标签页
2. 点击「📤 上传最终版纪要」
3. 选择修改后的最终版文件
4. 系统自动更新 reference 文件
5. 下次生成时会应用最新的 reference
```

## 常见问题

### Q: 生成的纪要不准确怎么办？
**A:** 可以：
1. 修改 `skill.md` 调整生成规则
2. 在「🧠 Skill 进化」标签页输入修改要求
3. 上传最终版纪要更新 reference

### Q: API 调用失败怎么办？
**A:** 请检查：
1. GLM_API_KEY 是否正确配置
2. API Key 是否有效（访问 https://open.bigmodel.cn 检查）
3. 网络连接是否正常

### Q: 如何查看生成历史？
**A:** 生成的纪要保存在 `outputs/` 目录，按时间戳命名，可以查看历史记录。

### Q: 是否支持批量生成？
**A:** 当前版本一次生成一个会议纪要，通过更换 inputs/ 目录中的文件可以批量处理。

### Q: 生成的纪要格式不对？
**A:** 检查：
1. `skill.md` 是否完整
2. reference 文件是否正确配置
3. 手写重点文件是否包含必要的会议信息
