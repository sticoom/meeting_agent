# 风格学习功能使用说明

## 🎯 功能概述

本系统实现了**持续学习风格模板**功能，通过对比初稿和最终稿，自动学习您的写作风格，让生成的会议纪要越来越像人写的。

---

## ✨ 核心特性

### 1. 持续学习
- 每次上传最终版纪要时，系统自动分析您的修改
- 提取风格规则并更新到模板
- 越用越智能，逐渐适应您的写作风格

### 2. Token 消耗可控
- 只在上传最终稿时调用学习 API
- 每次约消耗 3000 tokens
- 相比直接提供历史纪要节省约 75-80% tokens

### 3. 用户无感知
- 自动学习，无需手动配置
- 每次生成时自动应用最新风格

---

## 🚀 使用流程

### 第一步：生成初稿
1. 上传录音转写文件
2. 输入手写重点
3. 点击「🚀 一键生成会议纪要」
4. 系统生成初稿

### 第二步：人工修改
1. 人工修改初稿
2. 调整格式、内容、表达方式
3. 形成最终版纪要

### 第三步：上传最终稿
1. 在「📊 结果展示」标签页
2. 点击「📤 上传最终版纪要」
3. 系统自动：
   - ✅ 对比初稿和最终稿
   - ✅ 提取您的风格规则
   - ✅ 更新风格模板
   - ✅ 保存学习记录

### 第四步：下次生成
1. 重新上传新的录音转写文件
2. 系统会应用新学到的风格模板
3. 生成的纪要更接近您的写作风格

## 📊 Token 消耗分析

### 每次生成会议纪要
- 输入：录音转写（2000-3000 tokens）+ 手写重点（500-1000 tokens）+ 风格模板（500-1000 tokens）
- 输出：会议纪要初稿（2000-3000 tokens）
- **总计：约 5000-8000 tokens**

### 每次上传最终稿
- 输入：初稿（2000-3000 tokens）+ 最终稿（2000-3000 tokens）
- 输出：风格学习结果（500-1000 tokens）
- **总计：约 4500-7000 tokens**

### 对比方案
| 方案 | 每次生成 tokens | 学习调用 tokens | 10次总计 |
|------|----------------|----------------|----------|
| 提供历史纪要 | 20,000-30,000 | 0 | 200,000-300,000 |
| **持续学习** | 5,000-8,000 | 4,500-7,000 | 50,000-80,000 |

**节省约 75-80% 的 tokens！**

## 💡 使用技巧

### 1. 初次使用
- 风格模板为空，使用默认风格
- 生成后进行人工修改
- 上传最终稿，让系统学习

### 2. 持续优化
- 每次修改都是学习机会
- 保持一致的修改风格
- 系统会逐渐适应

### 3. 查看学习记录
- 学习记录保存在 `reference/03_风格学习记录.md`
- 可以查看每次学习的内容
- 了解系统的学习进度

### 4. 手动调整风格模板
- 如需快速调整，直接编辑 `reference/04_风格模板.md`
- 下次生成时会立即应用

## ⚠️ 注意事项

### 1. 风格模板需要初始学习
- 首次使用时，风格模板为空
- 需要通过上传最终稿进行学习
- 通常 2-3 次学习后效果显著

### 2. API Key 配置
- 风格学习需要 GLM-4 API Key
- 确保在 Secrets 中配置了 `GLM_API_KEY`
- 风格学习会消耗额外的 tokens

### 3. GitHub 同步
- 风格文件会自动同步到 GitHub
- 确保 GitHub Token 有 `repo` 权限
- 云端和本地保持一致
