## 2026-03-10 风格学习记录

### 新发现的写作风格

## 写作风格指南

### 1. 措辞特点
- 动词类："梳理"、"统筹推进"、"提炼"、"确保"、"优化"
- 名词类："流程"、"标签"、"合规性"、"风险"、"节点"
- 形容词类："关键核心"、"统一"、"良性"、"规范"
- 过于口语化的表达
- 模糊不清的词汇
- 情感色彩强烈的词汇
- "需进一步调研JCT税务合规风险及同行的合规操作模式"（专业术语）
- "要加大清库力度，加速出库，可利用当前较好的流量环境快速处理"（专业且正式的表达）
- "部门管理者要引导团队将个人目标、部门目标和组织目标结合起来"（正式的管理术语）
- "抓主要矛盾"、"抓核心、抓主要"（简洁有力的表达）
- "回归至2024年下半年之前持续盈利、稳步向好的发展轨道上来"（较为正式的表达）

### 2. 句子结构
- "目前仅有分Fnsku+站点维度的定位标签，缺乏SKU层级的全生命周期标签，导致数据分析困难，需梳理一套统一逻辑（例如：只要任一平台在售，该SKU即视为'在售'）。"（复合句，包含多个信息点和解释说明）
- "各部门负责人需重点关注春节假期前后的员工请假安排：对于部门内的关键核心岗位，若业务存在实际需求，需确保必要时有人在岗，全力保障假期前后各项业务正常运转。"（复合句，包含条件状语和目的状语）
- "冷总作为总负责人会深度参与，由雷佳和彭欢统筹推进流程优化项目落地，由雷佳、彭欢主导。"（并列结构，明确责任分工）

### 3. 表达习惯
- "冷总作为总负责人会深度参与，由雷佳和彭欢统筹推进流程优化项目落地，由雷佳、彭欢主导。"（明确责任分工）
- "各部门负责人需重点关注春节假期前后的员工请假安排：对于部门内的关键核心岗位，若业务存在实际需求，需确保必要时有人在岗，全力保障假期前后各项业务正常运转。"（条件状语和目的状语结合）
- "计划：优先利用深圳仓和云仓的已有库存发货，如外协仓能到货也可以一起发走，当这些SKU数量仍旧不能满足的情况下，可通过'sku组合/打包'的方式快速裂变出新SKU，以满足数量要求。"（策略与行动结合）

### 4. 重点强调方式
- "红线：寻源端后续严禁出现3位小数的报价单，统一保留2位小数。"（使用"红线"一词强调重要性）
- "我们解决问题的时候要抓主要矛盾，部门负责人管理团队很难面面俱到，不可能每个人都照顾到，所以我们要抓核心、抓主要，一个团队里面的核心就是表现优异的骨干人员，需要管理者特别关注。"（重复强调"抓主要"和"抓核心"）
- "此前已明确经营目标：一季度实现扭亏为盈，上半年达成稳定盈利。"（单独成段强调经营目标）

>
> **更新原则：**
> - 用最新的总结替代或补充旧的总结
> - **不删除过去 reference 中的重点内容**
> - 如遇特殊情况（如流程重大变更），可在保留旧内容的基础上添加说明

## 撰写注意事项

### 1. 去口语化
| 口语化表达 | 正式表达 | 备注 |
|----------|----------|------|
| 那个 | （删除或替换为连接词） | 填充词 |
| 话说 | （删除） | 填充词 |
| 说白了 | 换言之 / 具体来说 | 转换表达 |
| 死掉 | 失效 / 终止 | 严重口语 |
| 搞一下 | 推进 / 落实 | 轻率表达 |
| 弄一下 | 处理 / 执行 | 轻率表达 |
| 刚才说 | 前述 / 上述 | 时间参照 |
| 然后呢 | 此外 / 另外 | 连接词 |
| 就是这样 | 以上所述 | 总结语 |
| 就是说 | 即 / 也就是 | 解释说明 |
| 好多 | 大量 / 众多 | 数量表达 |
| 有点 | 略显 / 稍显 | 程度表达 |
| 挺 | 较为 / 相当 | 程度表达 |
| 嘛 | （删除） | 语气词 |
| 哦 | （删除） | 语气词 |
| 哈哈哈 | （删除） | 语气词 |

- 全文严禁出现口语化表达，统一使用高级管理语言

### 2. 逻辑分层
- 复杂逻辑强制使用 `1. 2. 3.` 递进
- 冷总的发言必须保留其商业哲理（如：平衡变量、回到现场）
- 使用清晰的层级结构
- 确保逻辑清晰，易于阅读

### 3. 主语明确
- 所有 Action Items 必须带有明确的责任人
- 示例：刘雨需、Alter负责、周立军负责
- 不得使用模糊的指代

### 4. 存疑高亮机制
- 对于金额、日期、具体责任人模糊不清的信息，绝对不准瞎编
- 必须使用醒目的占位符标出：`**[待确认：具体责任人/金额]**`
- 宁可留空用占位符，也不得擅自编造

## 数据准确性要求

### 1. 销售数据
- 必须明确销售额达成率
- 必须明确毛利额达成率
- 必须明确毛利率等关键指标
- 数据必须准确，不得模糊

### 2. 时间节点
- 所有事项都必须明确截止日期
- 日期格式统一为：YYYY-MM-DD
- 不得使用模糊的时间表达

### 3. 责任归属
- 每项行动项都必须明确责任人
- 责任人姓名必须准确
- 不得使用模糊的指代

### 4. 产品信息
- 产品名称必须准确
- 品类必须明确
- 数量等信息必须准确

## Reference 更新规则

### 更新时机
- 用户上传最终版会议纪要时

5. **不删除过去 reference 中的重点内容**
6. 如遇特殊情况，可在保留旧内容的基础上添加说明

## 特殊情况处理

### 1. 文件缺失
- inputs/ 文件夹为空或缺少必要文件时，提示用户上传
- 明确告知需要上传的文件类型和命名规范

### 2. 术语词典缺失
- 如 02_组织与术语词典.md 文件不存在，提示用户创建
- 提供术语词典模板
- 暂时不进行术语纠正，但提醒用户

### 3. 历史纪要缺失
- 如 reference/ 文件夹中没有历史纪要，提示用户上传
- 使用基本格式进行生成
- 提醒用户后续上传历史纪要以提升生成质量

### 4. 大量待确认项目
- 如存在大量待确认项目，总结性描述会议内容
- 在关键数据处使用占位符
- 提醒用户确认后重新生成

### 5. 冷总发言识别困难
- 如冷总在转写稿中身份不明确，主动向用户询问
- 提供可能的选项供用户确认
- 确保冷总发言的准确归属
